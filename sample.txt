전에 API 리스트업을 하면서 이것저것 적었더니 문서가 엄청 길어졌고, 그 결과 (그다지 보고 싶지 않게 생겼지만) 일 잘한 것 같아 보이는 문서가 만들어지게 되었다. 별 이견 없이 인프라는 누구, 아키텍처는 누구, API는 나, 이렇게 업무 분배가 되었다.

프로덕트에 필요한 API를 신입이 설계해 볼 수 있는 기회는 흔치 않을 것이다. 더군다나 API 가이드는 모두가 일하는 방식에 조금씩 영향을 미치지 않는가. 데이터 모델링이나 아키텍처를 구상한다고 해도 클라이언트 개발자가 참여하는 리뷰 시간을 갖지는 않을 것이다.

많은 사람들이 참고하는 문서인 만큼, 두근두근하는 마음으로 업무를 시작했다.

1. 기획 문서와 디자인 시안에서 요구하는 데이터를 정리해 보았다.
2. 각 데이터의 타입을 정하고 필드명을 지었다.
3. 사내 위키 문서로 정리하고 JSON 예시 응답까지 만들어 붙여두었다.
4. 모두에게 미팅 콜을 날리고 리뷰 시작!
